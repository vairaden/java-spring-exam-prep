> 4 золотых сигнала мониторинга. Типы метрик по Prometheus (Counter, Gauge, Histogram и Summary). Квантили и персентили.

## 4 золотых сигнала (Google SRE)

Базовый набор метрик для мониторинга сервиса:

1. **Latency (задержка)** — время обработки запроса. Важно отделять задержку **успешных** и
   **ошибочных** запросов (быстрая ошибка не должна маскировать проблему). Смотрят персентили.
2. **Traffic (трафик)** — нагрузка: запросов в секунду (RPS), сообщений в секунду и т.п.
3. **Errors (ошибки)** — доля/число неуспешных запросов (`5xx`, исключения, таймауты).
4. **Saturation (насыщенность)** — насколько загружены ресурсы (CPU, память, пул соединений,
   очередь). Показывает, близка ли система к пределу.

(Метод USE — Utilization/Saturation/Errors — для ресурсов; RED — Rate/Errors/Duration — для
сервисов; перекликаются с golden signals.)

## Типы метрик Prometheus

**Counter (счётчик)** — монотонно **возрастающее** значение (только вверх, сброс при
рестарте). Для подсчёта событий: число запросов, ошибок, отправленных сообщений. Анализируют
через `rate()` (скорость роста):
```
http_requests_total{status="200"}
```

**Gauge (измеритель)** — значение, которое **растёт и падает**. Текущее состояние: занятая
память, число соединений в пуле, длина очереди, температура.
```
jvm_memory_used_bytes
```

**Histogram (гистограмма)** — распределение значений по **корзинам (buckets)**. Считает,
сколько наблюдений попало в каждый интервал (`_bucket`), плюс `_sum` и `_count`. Квантили
вычисляются **на стороне Prometheus** (`histogram_quantile`). Корзины **агрегируются** между
инстансами — подходит для распределённых систем:
```
http_request_duration_seconds_bucket{le="0.1"}
```

**Summary (сводка)** — тоже про распределение, но **квантили считаются на стороне
приложения** (клиента) и публикуются напрямую. Точнее для одного инстанса, но **не
агрегируется** между инстансами (нельзя усреднить персентили). Дороже по CPU.

| | Histogram | Summary |
|---|---|---|
| Где считаются квантили | в Prometheus (из корзин) | в приложении |
| Агрегация между инстансами | да | нет |
| Гибкость квантилей | любые (по запросу) | фиксированные при настройке |

Для микросервисов обычно предпочитают **Histogram**.

## Квантили и персентили

**Квантиль** — значение, ниже которого лежит заданная **доля** наблюдений. **Персентиль** —
квантиль в процентах. Пример: **p95 latency = 200 мс** означает, что 95% запросов
обработаны быстрее 200 мс.

Зачем не среднее: среднее **скрывает хвосты**. Медленные 1–5% запросов (p95/p99) часто и
есть проблема пользователей, а average их «размазывает». Поэтому SLO задают по персентилям
(p50/p90/p95/p99), а не по average.
