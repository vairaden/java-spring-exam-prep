> Фреймворк. Отличие библиотеки от фреймворка. Преимущества и недостатки использования фреймворков. Что такое Spring. Что такое Spring Boot.

## Что такое фреймворк

**Фреймворк** — это каркас приложения: набор библиотек, соглашений и точек расширения,
который задаёт структуру и берёт на себя управление потоком выполнения. Разработчик
пишет код, который встраивается в этот каркас, а фреймворк вызывает его в нужный момент.

## Отличие библиотеки от фреймворка

Ключевое отличие — **инверсия управления (Inversion of Control)** и принцип
«Голливуда»: *«Не звоните нам — мы позвоним вам»*.

| | Библиотека | Фреймворк |
|---|---|---|
| Кто кого вызывает | **Вы** вызываете код библиотеки | **Фреймворк** вызывает ваш код |
| Поток управления | У вашего кода | У фреймворка |
| Пример | Apache Commons, Guava, Jackson | Spring, Hibernate, JUnit |

Библиотека — это набор готовых функций, которые вы дёргаете когда захотите. Фреймворк
диктует архитектуру: вы заполняете «дырки» (реализуете контроллеры, сервисы), а он
оркестрирует их вызов.

## Преимущества фреймворков

- **Скорость разработки** — готовые решения типовых задач (DI, транзакции, веб, безопасность).
- **Единые соглашения** — код одинаково структурирован, легче входить новым разработчикам.
- **Меньше boilerplate** — рутина (управление ресурсами, сериализация, пулы) уже решена.
- **Сообщество и экосистема** — документация, обкатанные best practices, безопасность.

## Недостатки фреймворков

- **Кривая обучения** — нужно знать «как принято» во фреймворке.
- **Потеря контроля / «магия»** — поведение скрыто за аннотациями и автоконфигурацией.
- **Привязка (lock-in)** — миграция на другой стек дорогая.
- **Оверхед** — лишние абстракции и память по сравнению с «голым» кодом.

## Что такое Spring

**Spring Framework** — модульный фреймворк для Java-приложений, ядром которого является
**IoC-контейнер** (управление бинами и внедрение зависимостей). Поверх ядра строятся
модули: Spring MVC (веб), Spring Data (БД), Spring Security, Spring AOP и др. Spring
позволяет писать слабосвязанный, тестируемый код, делегируя контейнеру создание и
связывание объектов.

## Что такое Spring Boot

**Spring Boot** — надстройка над Spring, убирающая ручную конфигурацию:

- **Автоконфигурация** (`@EnableAutoConfiguration`) — настраивает бины по тому, что есть в classpath.
- **Стартеры** (`spring-boot-starter-web` и т.д.) — наборы согласованных зависимостей.
- **Встроенный сервер** (Tomcat/Jetty/Undertow) — приложение запускается как обычный `jar` (`java -jar`).
- **Удобная конфигурация** через `application.yaml`/`application.properties` и профили.
- **Production-ready** возможности (Actuator: метрики, health-checks).

Кратко: Spring даёт IoC и модули, Spring Boot делает их запуск «из коробки» без XML и ручной настройки.
